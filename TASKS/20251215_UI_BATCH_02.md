Batch 02 目标（只做交互反馈）
========================
主题切换不是本批重点（Batch 01 已有主题参数），但本批要实现：当主题 palette 更新时，所有控件的交互高亮/发光都跟随 ThemePalette.accent/glow。

本批只改“控件外观与交互反馈层”，不要改布局，不要改 APVTS 参数定义，不要改控件数量与摆放，不要改音频逻辑。

我们要实现的交互反馈与 Web 版一致：
1) Knob（Rotary Slider）
- 指示点（indicator dot）平时是灰色；拖拽/hover 时变为当前主题 accent
- 拖拽/hover 时指示点周围出现 glow（类似 CSS boxShadow: 0 0 5px accent）
- 角度范围必须是 -145° 到 +145°（和 Web 版 Knob.tsx 对齐）
- dot 的位置必须通过三角函数计算（cos/sin），而不是旋转整张图

2) Fader（Linear Vertical Slider）
- cap line（滑块帽的横线）永远是主题 accent
- fill（填充条）在拖拽/hover 时更亮/更实（accent alpha 更高），非激活时更淡
- 拖拽/hover 时 cap 或 fill 可有轻微 glow（可选，但建议做）

3) 主题 palette 注入
- LookAndFeel 持有 ThemePalette，并提供 setPalette(const ThemePalette&)
- Editor 在主题变化时调用 lookAndFeel.setPalette(palette) 并刷新
